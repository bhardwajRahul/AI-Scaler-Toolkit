# LLM Wiki Pattern

The LLM Wiki Pattern is a knowledge-base construction methodology proposed by Andrej Karpathy. Its core idea is to use an LLM to automatically convert source material into a structured, interconnected Markdown knowledge base.

## Core Concepts

### Difference from Traditional RAG

**Traditional RAG (Retrieval-Augmented Generation):**
```
query → search chunks → answer → forget
```
- Each query re-retrieves and re-processes
- Knowledge does not accumulate
- No persistent knowledge structure

**LLM Wiki Pattern:**
```
sources → compile → wiki → query → save → richer wiki → better answers
```
- Compiles knowledge into a persistent wiki
- Concept pages link to one another
- Query results can be saved and enrich the knowledge base

## Architectural Layers

### Layer 1: Raw Sources
- An immutable collection of source documents
- Includes articles, papers, images, data files, and more
- The LLM only reads, it does not modify

### Layer 2: The Wiki
- A directory of Markdown files generated by the LLM
- Contains summaries, entity pages, concept pages, comparative analyses, and more
- The LLM fully owns this layer: creating pages, updating content, maintaining cross-links

### Layer 3: The Schema
- Defines the file structure and naming conventions
- Standardizes the tag system
- Ensures the consistency of the knowledge base

## Workflow

1. **Ingest**: Obtain source material and save it to the raw/ directory
2. **Compile**: The LLM analyzes the sources, extracts concepts, and creates/updates wiki pages
3. **Query**: The user queries the knowledge base, and the LLM answers based on the compiled knowledge
4. **Save**: Valuable query results are saved as new wiki pages
5. **Lint**: Periodically check the quality of the knowledge base, finding and fixing issues

## Implementation Tool

### llm-wiki-compiler
A Node.js tool that implements the complete LLM Wiki Pattern:
- Supports multiple formats: HTML, PDF, images, text files
- Automatically extracts concepts and creates wiki pages
- Builds cross-links between pages
- Provides query and compile functions
- Supports Obsidian-compatible format

```bash
npm install -g llm-wiki-compiler
export ANTHROPIC_API_KEY=sk-ant-...
llmwiki quickstart ./notes.md
llmwiki compile
```

## Key Features

- **Automated**: The LLM is responsible for organizing, archiving, and maintaining
- **Cumulative**: The knowledge base becomes more valuable as it grows over time
- **Queryable**: Structured knowledge supports efficient querying
- **Traceable**: Every paragraph has a source citation marker
- **Interconnected**: Rich cross-links are established between concept pages
